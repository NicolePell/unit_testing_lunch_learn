Organising Tests

Arrange. Act. Assert.


Arrange.

Set the test system state by creating objects, interacting with them, calling other APIs, and so on.

Act.

Exercise the code you want to test, usually by calling a single method.

Assert.

Verify the code behaved the way you expected.

This can be checking the return value of the exercised code, or the new state of any objects involved. It can also be verifying the interactions between the tested and other objects took place.


Test Behaviour rather than Methods

Focus on the behaviour of your class, not on testing the individual methods.

Focused, Single-Purpose Tests

Sometimes you might be tempted to combine tests to avoid repeated setup, but doing this loses the benefit of test isolation that JUnit provides.

Immediately know what behaviour went wrong when an assertion fails.

Minimise the amount of sleuthing needed to decipher a failed test.

Ensure all cases get executed.
